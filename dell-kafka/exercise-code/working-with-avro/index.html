working-with-avro

Description

Files related to the "Working with Avro" exercise.

Verify that avro-tools is in your path

Run which avro-tools. If this does not return a path, install the package using command below

sudo yum install avro-tools

Import the Project into Eclipse

  1. In Eclipse, click the File menu, and then click the Import... item
  2. Expand the Maven folder
  3. Select Existing Maven Projects, and then click the Next button
  4. Click the Browse... button
  5. Navigate to find the current project's directory, select it, and then click the OK button
  6. Click the Finish button on the dialog

Ignore the warning that appears after import, which says that Eclipse doesn't have a marketplace entry for avro-maven-plugin. Click the Finish button on this dialog to dismiss it, and then click OK on the warning dialog that says that continuing will result in build errors, as they won't affect the exercise.

Define the Avro Schema

Open the activation.avsc file under src/main/resources in the IDE. The Avro schema has a comment that points out where to begin filling in the fields.

Define the following fields in the Avro schema :

  1. A field named timestamp of type long
  2. A field named customer_id of type int
  3. A field named device_id of type string
  4. A field named phone_number of type string
  5. A field named model of type string

If you need help, the activation.avsc_solution file shows the text of solution.

Observe How Maven Generates Java Code from the Avro Schema

The Maven pom.xml file in this exercise and the next one are configured to automatically create Java classes from the Avro schemas in your project. Take a moment to review the pom.xml file and examine the content of the build element near the bottom, which uses the avro-maven-plugin plugin to generate Java source code from Avro schemas. In this case, it expects to find Avro schemas in the src/main/resources directory and generates Java code to the target/generated-sources directory.

Because the avro-maven-plugin plugin is a plugin, not a dependency, we had to define an additional element (pluginRepositories) in the pom.xml file that specified the Cloudera repository where Maven will find the plugin.

Finally, because the Avro version is used multiple times in the pom.xml file, we defined a variable (avro.version) at the top of the file to hold that value, which we reference throughout the file.

You can use this as a guide for Avro projects you might create later.

Generate the Java Code from the Avro Schema

Run the Maven command below to generate Java source that corresponds to the Avro schema you just created:

mvn generate-sources

You could also do this from the IDE by right-clicking the pom.xml file, choosing Run as... from the context menu, and clicking on the Maven generate-sources item.

Add the Generated Source to the IDE's Build Path

Although the previous step generated Java code from the Avro schema, Eclipse does not yet consider it part of the project. Follow these steps to add it:

  1. Right-click the project in Eclipse and choose Refresh
  2. Right-click the project in Eclipse again, point to the Build Path submenu, and then click Configure Build Path...
  3. Click the Source tab
  4. Click the Add Folder button
  5. Expand the target folder, check the box to the left of the generated-sources subfolder, and then click the OK button
  6. Click the Apply and Close button

Next, have a quick look at the generated code, which you can find within the target/generated-sources folder that now appears below the project's icon in Eclipse. Fortunately, you'll only need to interact with the generate code by using its API!

Run the Unit Test

Open the com.cloudera.training.kafka.ActivationTest unit test class. Since it depends on Java code generated by Avro, most of its code was commented out so as not to cause compilation problems. Uncomment it now, and then run it. If you have properly defined the Avro schema, the test should pass.

Write Avro Data to a File for Inspection

Open the com.cloudera.training.kafka.ActivationCreator class in your IDE and uncomment out the code (since it depends on the Avro code generation, it was commented out so as not to cause compilation problems before that code was generated).

This class shows two different ways of creating an Activation instance from the generated code.

After looking at the uncommented code, run the class' main method, either in Eclipse or by using the command shown below:

mvn exec:java \
    -Dexec.mainClass="com.cloudera.training.kafka.ActivationCreator"

It will create a file at /home/training/Desktop/activations.avro.

View the Data in the Avro File

avro-tools tojson /home/training/Desktop/activations.avro

View the Avro File's Embedded Schema

avro-tools getschema /home/training/Desktop/activations.avro
